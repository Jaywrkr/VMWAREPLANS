    vSphere Foundation

    • Virtualización de cómputo con vSphere 9
    • vCenter Server para gestión centralizada
    • vSphere Kubernetes Service (VKS) básico
    • Aria Suite Lifecycle Manager
    • vSAN OSA (All-Flash opcional)
    • vSphere Distributed Switch (vDS)
    HypervisorvCenter vDSVKS básico

    VCFE

    VCF Edge (Nube Privada Edge)

    • Todo lo de VVF más stack completo de VCF
    • NSX para redes definidas por software
    • vSAN ESA (Express Storage Architecture)
    • VKS Enterprise + ArgoCD Operator
    • VCF Automation (hasta 5 hosts)
    • VCF Operations (Aria Operations)
    NSXvSAN ESA VKS Enterprise≤5 hosts

    VCF

    VMware Cloud Foundation

    • Plataforma de nube privada completa
    • NSX Enterprise + Gateway Firewall
    • vSAN ESA sin límite de hosts
    • VKS Enterprise + Harbor Registry
    • VCF Automation ilimitado
    • VCF Operations + Log Intelligence
    • Private AI Foundation (GPU workloads)

    Casos de uso · Antes y Después

    🏢

    Empresa con 20 servidores físicos

    VVF
    Antes
    • 20 servidores físicos, 1 app por servidor
    • CPU promedio al 8% de uso
    • Mantenimiento: ventana nocturna por servidor
    • Fallo de hardware = downtime de app
    Con VVF
    • 3-4 hosts ESXi con 20 VMs cada uno
    • CPU al 60-70%, ahorro real de hardware
    • Parches sin downtime con vMotion
    • HA reinicia VMs en segundos ante fallo
    🔒

    Banco con requisitos de microsegmentación

    VCFE
    Antes
    • Firewall perimetral único: norte-sur solo
    • Tráfico este-oeste libre entre VMs
    • Un compromiso puede propagarse libremente
    • VLANs como único aislamiento de red
    Con VCFE
    • NSX DFW: cada VM tiene su propio firewall
    • Microsegmentación: tráfico este-oeste controlado
    • Ransomware contenido en la VM infectada
    • Políticas por etiqueta, no por IP
    🐳

    Equipo DevOps con Kubernetes

    VCF
    Antes
    • Clusters K8s manuales en servidores separados
    • Imágenes en Docker Hub público
    • Deploy manual: "funciona en mi máquina"
    • Sin visibilidad de quién despliega qué
    Con VCF
    • TKG Enterprise: clusters en minutos
    • Harbor Registry privado con escaneo de CVEs
    • ArgoCD: GitOps automático desde la rama main
    • Namespaces con cuotas y auditoría completa
    🤖

    Farmacéutica con IA confidencial

    VCF
    Antes
    • ChatGPT/Claude para análisis de datos internos
    • Datos de investigación enviados a la nube pública
    • Riesgo de fuga de IP y violación regulatoria
    • Sin control del modelo ni del contexto
    Con VCF
    • LLaMA 3 on-premises con Private AI Foundation
    • Datos nunca salen del datacenter propio
    • Cumplimiento GxP, GDPR y regulación farmacéutica
    • RAG sobre documentos internos de investigación
    📊

    IT que tarda semanas en aprovisionar

    VCF
    Antes
    • Ticket a IT → espera 2 semanas por una VM
    • Configuración manual: errores frecuentes
    • Sin visibilidad de costos por equipo
    • Infraestructura no reproducible ni versionada
    Con VCF
    • Portal self-service: ambiente listo en 20 min
    • Blueprints con Terraform: reproducibles y en Git
    • Chargeback por equipo desde VCF Operations
    • Aprobaciones automáticas por tipo de recurso
    🌐

    Sucursal remota con 3 hosts

    VCFE
    Antes
    • Servidores físicos independientes en la sucursal
    • Sin nube privada: depende de VPN a sede central
    • Sin SDN ni storage hiperconvergido local
    • Gestión manual por técnico en sitio
    Con VCFE
    • Stack VCF completo en 3-5 hosts (edge)
    • vSAN ESA + NSX local, sin depender de WAN
    • Gestión remota desde sede central vía SDDC Manager
    • Alta disponibilidad local ante fallos de red WAN

    Rutas de migración / upgrade

    Desde vSphere Foundation / vSphere EP

    • → VVF (migración directa)
    • → VCFE (si necesita NSX/vSAN)
    • → VCF (stack completo)

    Desde vCloud Suite

    • → VCF (equivalente funcional)
    • → VVF + add-ons (reducción de scope)

    Desde Aria Suite (standalone)

    • → VCF (Aria queda incluido)
    • → VCFE (Operations incluido)

    Desde VCF 4.x / 5.x

    • → VCF 9.1 (upgrade in-place)
    • → Requiere VCF 5.2+ como base

    Desde NSX-T standalone

    • → VCFE (NSX incluido)
    • → VCF (NSX Enterprise incluido)

    Desde vSAN standalone

    • → VCFE (vSAN ESA incluido)
    • → VCF (vSAN ESA + Max disponible)
    📋 Broadcom recomienda revisar la herramienta de compatibilidad oficial antes de cualquier upgrade. Las rutas de migración dependen de la versión actual y del hardware certificado HCL.
