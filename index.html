Los tres planes VMware

VVF

vSphere Foundation

  • Virtualización de cómputo con vSphere 9
  • vCenter Server para gestión centralizada
  • vSphere Kubernetes Service (VKS) básico
  • Aria Suite Lifecycle Manager
  • vSAN OSA (All-Flash opcional)
  • vSphere Distributed Switch (vDS)
Hypervisor vCenter vDS VKS básico

VCFE

VCF Edge (Nube Privada Edge)

  • Todo lo de VVF más stack completo de VCF
  • NSX para redes definidas por software
  • vSAN ESA (Express Storage Architecture)
  • VKS Enterprise + ArgoCD Operator
  • VCF Automation (hasta 5 hosts)
  • VCF Operations (Aria Operations)
NSX vSAN ESA VKS Enterprise ≤5 hosts

VCF

VMware Cloud Foundation

  • Plataforma de nube privada completa
  • NSX Enterprise + Gateway Firewall
  • vSAN ESA sin límite de hosts
  • VKS Enterprise + Harbor Registry
  • VCF Automation ilimitado
  • VCF Operations + Log Intelligence
  • Private AI Foundation (GPU workloads)
Full Stack NSX Enterprise Private AI Sin límite

¿Cuál plan es para ti?

🏢

Virtualización clásica on-prem

Empresa que migra desde vSphere tradicional, sin necesidad de SDN ni nube privada completa.

Recomendado: VVF
🌐

Nube privada en sede remota

Oficina o sucursal con hasta 5 hosts que necesita stack completo de VCF en espacio reducido.

Recomendado: VCFE
🤖

Cargas de trabajo de IA / GPU

Proyectos de IA generativa que requieren Private AI Foundation y acceso a NVIDIA AI Enterprise.

Recomendado: VCF
🐳

Kubernetes en producción

Plataformas de contenedores empresariales con Harbor Registry, ArgoCD y red NSX.

Recomendado: VCF
🔒

Seguridad y microsegmentación

Entornos que requieren NSX Distributed Firewall y políticas de microsegmentación avanzadas.

Recomendado: VCFE o VCF
📊

Multi-cloud con automatización

Organizaciones que necesitan VCF Automation para self-service de infraestructura a escala.

Recomendado: VCF
💰

Presupuesto limitado, funciones clave

Empresa que quiere hypervisor enterprise, vCenter y Kubernetes básico al menor costo.

Recomendado: VVF
🔄

Migración desde Aria / vCloud Suite

Clientes con licencias vCloud Suite, Aria Suite o vSphere EP que buscan transicionar a SKU 9.1.

Recomendado: VCF

Rutas de migración / upgrade

Desde vSphere Foundation (VSF) / vSphere EP

  • → VVF (migración directa)
  • → VCFE (si necesita NSX/vSAN)
  • → VCF (stack completo)

Desde vCloud Suite

  • → VCF (equivalente funcional)
  • → VVF + add-ons (reducción de scope)

Desde Aria Suite (standalone)

  • → VCF (Aria queda incluido)
  • → VCFE (Operations incluido)

Desde VCF 4.x / 5.x

  • → VCF 9.1 (upgrade in-place)
  • → Requiere VCF 5.2+ como base

Desde NSX-T standalone

  • → VCFE (NSX incluido)
  • → VCF (NSX Enterprise incluido)

Desde vSAN standalone

  • → VCFE (vSAN ESA incluido)
  • → VCF (vSAN ESA + Max disponible)
📋 Broadcom recomienda revisar la herramienta de compatibilidad oficial antes de cualquier upgrade. Las rutas de migración dependen de la versión actual y del hardware certificado HCL.
